1. 工程化：是一种思维，涵盖开发到上线的各个环节（开发、分支管理、构建、部署、自动化测试、性能优化等等）
	只要在各自的环节上能提升效率，都属于工程化的范畴（开发 构建 部署 性能）
2. 模块化：
	发展过程：一份大的 js 拆分为多份 js => 命名空间（解决全局变量污染）=> 闭包（解决外部能够修改命名空间内的值，数据保护）
	=> commonjs（解决引入顺序，模块之间的依赖关系）exports = module.exports，require引入，注意：服务于服务端，运行时同步加载
	=> amd (requirejs)、cmd (sea.js) 服务于浏览器端，也是运行时才能确定导入的内容
	=> es6 module 浏览器和服务端通用，这个在编译的时候就能确定模块之间的依赖关系，也就是编译时加载，利于 treeshaking
3. 组件化：是指将某一个小功能包含（template style script）封装起来，高内聚低耦合，便于复用
